# Sonalake Design System

[Sonalake Design System (SDS)](https://github.com/sonalake/design-system) is a reusable React component library that helps **Internal Sonalake Project** contributors build UIs faster. The goal is to make the building of UIs more durable, productive and satisfying, inline with the [Design System](). An emphasis on Accessibility and Internationalisation is included in the development of the libaray.

## Install

via yarn:

```
yarn add @sonalake/design-system
```

via npm:

```
npm install @sonalake/design-system
```

## Dependencies

The SDS relies on a number of open source packages, listed below:

- [tsdx](https://tsdx.io/) - _Typescript package development_
- [radix ui](https://www.radix-ui.com/) - _Accessible component library_
- [clsx](https://github.com/lukeed/clsx) - _className util package_
- [date-fns](https://date-fns.org/) - _Date formatting package_
- [lodash](https://lodash.com/) - _Utility functions package_
- [react-icons](https://react-icons.github.io/react-icons/) - _React icons package_
- [react-intl](https://formatjs.io/docs/react-intl/#the-react-intl-package) - _Internationalisation library_
- [react-table](https://react-table.tanstack.com/) - _React table package_
- [tailwindcss](https://tailwindcss.com/) - _Utility-first CSS framework_

## Architecture

A list of useable components can be seen in the navigation under Atoms and Molecules. SDS follows [Atomc Design Principles](https://bradfrost.com/blog/post/atomic-web-design/).


## Styling

If you wish you can include the SDS tailwind config in your own projects configuration.

```javascript
// tailwind.config.js
module.exports = {
  content: ['./node_modules/@sonalake/design-system/src/**/*.{js,jsx,ts,tsx}'],
  presets: [require('@sonalake/design-system/dist/tailwind.config.js')],
};
```

## Usage

To use SDS in your own internal React project, it is recommended to include the DesignSystemProvider.

```typescript
import { DesignSystemProvider } from '@sonalake/design-system';

<DesignSystemProvider language="en" locales={{ en: STORYBOOK_LOCALES }}>
  // Your app goes here
</DesignSystemProvider>;
```

This will give you access to the following benefits of the SDS:

- Internationalisation
- Formatting
- Notifications
